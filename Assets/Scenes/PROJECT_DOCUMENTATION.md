# PathFinder Simulator 3D — 프로젝트 문서

> 이 문서는 보고서 작성을 위한 프로젝트 요약 자료입니다. 프로젝트의 목적, 구조, 구현된 기능, 각 스크립트의 역할, 알고리즘 상세를 정리합니다.

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | PathFinder Simulator 3D |
| 엔진 | Unity 6 |
| 언어 | C# |
| 소속 | 고려대학교 세종캠퍼스 S-CURT 2026-1 |
| 분야 | 경로 탐색(Pathfinding) 알고리즘 시각화 및 성능 비교 시뮬레이터 |

### 프로젝트 목적

이 프로젝트는 대표적인 그래프 탐색 및 경로 탐색 알고리즘 7종을 **3D 그리드 위에서 실시간으로 시각화**하고, 각 알고리즘의 **성능을 정량적으로 측정·비교**할 수 있는 교육·연구용 시뮬레이터입니다.

핵심 목표는 다음과 같습니다.

- 알고리즘이 노드를 탐색해 나가는 과정(방문 순서, 확산 형태)을 눈으로 확인
- 동일한 맵/조건에서 알고리즘 간 성능(계산 시간, 방문 노드 수, 메모리, 경로 길이)을 비교
- 사용자가 직접 벽을 그리거나 랜덤 맵을 생성하고, 시작점·목표점을 옮기며 실험
- 에이전트(Player)가 계산된 경로를 따라 실제로 이동하는 모습을 관찰
- 이동 중 장애물이 추가되면 즉시 경로를 재계산하는 동적 재탐색 확인

---

## 2. 구현된 핵심 기능 요약

1. **7종 경로 탐색 알고리즘** — A*, Dijkstra, BFS, DFS, Greedy Best-First, D*, D* Lite
2. **탐색 과정 애니메이션** — 방문 노드를 순서대로 채색한 뒤 최종 경로를 강조 표시
3. **에이전트 경로 추종 이동** — Player 오브젝트가 계산된 경로를 따라 부드럽게 이동
4. **동적 재탐색** — 이동 중 벽 추가/삭제, 목표 이동 감지 시 즉시 경로 재계산
5. **인터랙티브 맵 편집** — 마우스 드래그로 벽 그리기/지우기, 시작·목표점 이동
6. **랜덤 맵 생성** — 벽 밀도 기반 랜덤 배치 + BFS로 경로 존재 보장
7. **성능 통계 패널** — 마지막 탐색의 방문 수/경로 길이/시간/메모리 표시
8. **성능 누적 대시보드** — 같은 맵에서 알고리즘별 최신 성능을 표로 누적
9. **일괄 비교 벤치마크** — 7개 알고리즘을 한 번에 실행하여 표로 비교, 최우수 항목 강조
10. **알고리즘 선택 UI** — 드롭다운 및 개별 버튼으로 알고리즘 전환

---

## 3. 시스템 아키텍처

프로젝트는 크게 **AI(탐색 코어)**, **Core(맵 생성)**, **UI(시각화·상호작용)** 세 계층으로 구성됩니다.

```
Assets/Scripts/
├── AI/
│   ├── GridNode.cs        # 그리드 셀 데이터 단위(노드)
│   ├── GridManager.cs     # 그리드 생성·시각화·편집·이웃 조회
│   ├── MinHeap.cs         # 우선순위 큐(이진 최소 힙) 자료구조
│   ├── PathFinderAI.cs    # 7종 알고리즘 구현 + 탐색 애니메이션 + 메트릭 수집
│   └── PlayerAI.cs        # 에이전트 경로 추종 이동
├── Core/
│   └── MapGenerator.cs    # 랜덤 벽 배치 + 경로 보장
└── UI/
    ├── AlgorithmSelector.cs     # 알고리즘 선택 UI
    ├── AlgorithmStatsPanel.cs   # 마지막 탐색 통계 패널
    ├── AlgorithmDashboard.cs    # 알고리즘별 성능 누적 대시보드
    └── AlgorithmBenchmark.cs    # 7종 일괄 비교 벤치마크
```

### 데이터 흐름

```
[사용자 입력]
   │  (벽 편집 / 시작·목표 이동 / 랜덤 맵)
   ▼
GridManager  ──(그리드·이웃·좌표변환 제공)──►  PathFinderAI
   ▲                                              │
   │ (셀 채색 요청)                                │ (경로·메트릭 산출)
   │                                              ▼
   │                             ┌──────────────────────────────┐
   │                             │  PlayerAI (경로 추종 이동)     │
   │                             │  StatsPanel / Dashboard /      │
   └─────────────────────────────┤  Benchmark (성능 표시)         │
                                 └──────────────────────────────┘
```

- **GridManager**가 그리드 데이터의 단일 소스이며, 이웃 노드·좌표 변환·셀 채색을 담당합니다.
- **PathFinderAI**가 탐색 알고리즘을 실행하고 성능 메트릭을 산출합니다.
- **PlayerAI**는 PathFinderAI에서 다음 이동 좌표를 받아 에이전트를 이동시킵니다.
- **UI 계층**은 PathFinderAI의 메트릭과 이벤트를 구독하여 성능을 표시합니다.

## 4. 스크립트별 상세

### 4.1 AI 계층

#### GridNode.cs — 그리드 셀 데이터 노드
그리드의 각 셀 하나를 표현하는 순수 데이터 클래스입니다.

- **좌표 정보**: `row`, `col`, `worldPosition`(월드 공간 위치)
- **탐색 상태**: `isWall`(장벽), `isVisited`(방문 완료), `isPath`(경로 포함), `isOpen`(열린 목록)
- **비용 필드**: `gCost`(시작점→현재 실제 비용), `hCost`(현재→목표 휴리스틱), `FCost`(= G + H), `rhs`(D* Lite 전용 선견 비용)
- **역추적**: `parent`(경로 복원용 부모 노드)
- **시각화**: `cellObject`(셀 GameObject 참조)
- `ResetAll()`(장벽 포함 전체 초기화), `ResetSearchState()`(장벽 유지, 탐색 상태만 초기화) 제공

#### MinHeap.cs — 우선순위 큐(이진 최소 힙)
비교 함수를 주입받는 제네릭 이진 최소 힙 자료구조입니다. (MonoBehaviour 아님)

- 매 반복마다 `List.Sort`를 호출하는 대신 `Push`/`Pop`을 **O(log n)** 으로 수행
- 비교 함수를 바꿔 A*(F 비용), Dijkstra(G 비용), Greedy(H 비용), D*(G 비용)에 재사용
- `Push`, `Pop`, `Count` API 제공, 내부적으로 SiftUp/SiftDown 구현

#### GridManager.cs — 그리드 생성·시각화·편집 매니저
3D 그리드의 생성, 렌더링, 사용자 편집을 총괄하는 핵심 매니저입니다.

- **그리드 생성**: 기본 32열 × 20행, Cube 프리미티브를 코드로 생성하여 셀 시각화 (`BuildGrid`)
- **셀 채색**: 상태별 색상(일반/벽/시작/목표/방문/열림/경로)을 `MaterialPropertyBlock`으로 적용해 머티리얼 인스턴스 생성 방지, URP(`_BaseColor`)·Built-in(`_Color`) 양쪽 대응
- **마우스 편집**: Unity Input System 기반, 레이캐스트로 셀을 찾아 드래그로 벽 토글 (`HandleMouseInput`)
- **시작·목표 스냅**: startTransform/endTransform 이동을 감지해 가장 가까운 셀로 스냅 (`UpdateStartEndSnap`)
- **이웃 조회**: 상하좌우 4방향 이웃 반환, 장벽·범위 밖 제외 (`GetNeighbors`)
- **좌표 변환**: 월드↔그리드 인덱스 변환 (`GetNodeFromWorldPosition`, `GetWorldPosition`)
- **변경 감지 신호**: `WallsDirty` 플래그와 `MapVersion` 카운터로 재계산·대시보드 초기화 트리거
- **랜덤 위치**: 벽이 아닌 노드 중 시작·목표를 랜덤 지정 (`RandomizeStartEnd`)
- **기즈모 시각화**: Scene 뷰에서 그리드 경계·셀 상태를 기즈모로 표시

#### PathFinderAI.cs — 경로 탐색 코어 (프로젝트의 핵심)
7종 알고리즘을 구현하고, 탐색 애니메이션과 성능 메트릭을 담당합니다.

- **알고리즘 열거형**: `Algorithm { AStar, Dijkstra, BFS, DFS, GreedyBestFirst, DStar, DStarLite }`
- **메트릭 클래스** `SearchMetrics`: 알고리즘명, 성공 여부, 방문 노드 수, 경로 길이, 계산 시간(ms), 최대 열린 목록 크기(피크 메모리 지표)
- **탐색 애니메이션**: `PrepareAndAnimate` 코루틴이 방문 노드를 순서대로 채색한 뒤 경로를 강조 (`AnimateSearch`)
- **다음 좌표 반환**: `SetNextPOS`가 PlayerAI에 다음 이동 목표를 반환하며, 목표 변경·벽 변경·경로 소진 시 자동 재계산
- **동적 재탐색**: `Update`에서 `GridManager.WallsDirty`를 감지해 재계산 플래그 설정, `RequestRecalculate`로 외부 강제 재계산
- **일괄 실행**: `RunAllAlgorithms`가 동일 시작·목표점으로 7종을 모두 실행하여 메트릭 목록 반환
- **완료 이벤트**: `OnSearchCompleted` 이벤트로 대시보드에 전체 탐색 완료 통지
- **성능 측정**: `Stopwatch`로 계산 시간 측정, 방문 순서·최대 프론티어 크기 기록
- **공용 유틸**: 맨해튼 거리 휴리스틱(`Heuristic`), 경로 역추적(`TracePath`), 비용 기울기 경로 추출(`ExtractGradientPath`)

#### PlayerAI.cs — 에이전트 경로 추종 이동
Player 오브젝트를 계산된 경로를 따라 이동시킵니다.

- **이동 제어**: `StartMovement`(현 위치에서 이동), `StopMovement`(정지 + 시각화 제거), `ResetToStart`(시작점 복귀 후 재이동)
- **애니메이션 연동**: 이동 전 `AnimateThenMove`로 탐색 확산 애니메이션을 먼저 재생한 뒤 이동 시작
- **경로 추종**: `Move` 코루틴이 `SetNextPOS`를 반복 호출하며 다음 좌표로 이동, `MoveToPosition`으로 부드럽게 보간·회전
- **동적 대응**: 이동 중 `IsPathDirty`(벽 변경)나 목표 이동을 감지하면 즉시 종료하고 재경로 계산에 위임
- **경로 없음 처리**: 유효 경로가 없으면 잠시 대기 후 재시도(벽이 치워지거나 목표가 바뀔 때까지)

### 4.2 Core 계층

#### MapGenerator.cs — 랜덤 맵 생성
그리드 위에 벽 프리팹을 랜덤 배치하여 맵을 생성합니다.

- **벽 밀도 기반 랜덤 배치**: `wallDensity`(0.0~0.8) 확률로 셀을 벽으로 지정
- **경로 보장**: 배치 후 BFS로 시작→목표 경로 존재를 확인, 실패 시 최대 `maxRetryCount`회 재시도 (`HasPathBFS`)
- **시작·목표 보호**: 시작점·목표점 셀에는 벽을 생성하지 않음
- **맵 초기화**: 생성된 벽 오브젝트 전체 제거 및 그리드 초기화 (`ClearMap`)
- 에디터 우클릭(`[ContextMenu]`)과 외부 버튼 양쪽에서 호출 가능

### 4.3 UI 계층

#### AlgorithmSelector.cs — 알고리즘 선택 UI
- TMP_Dropdown으로 7종 알고리즘 선택, 선택값을 PathFinderAI에 적용
- 개별 버튼용 메서드(`SelectAStar`, `SelectDijkstra` 등)도 제공하여 버튼 OnClick 직접 연결 지원
- 알고리즘 이름은 `PathFinderAI.GetAlgorithmName`으로 단일화

#### AlgorithmStatsPanel.cs — 마지막 탐색 통계 패널
- `PathFinderAI.LastMetrics`를 폴링하여 마지막 탐색 결과 표시
- 표시 항목: 알고리즘명, 성공/실패, 방문 노드 수, 경로 길이, 계산 시간(ms), 최대 메모리
- 패널 On/Off 토글 지원

#### AlgorithmDashboard.cs — 성능 누적 대시보드
- `PathFinderAI.OnSearchCompleted` 이벤트를 구독하여 알고리즘별 성능을 표로 누적
- 알고리즘당 최신 1레코드만 유지, 같은 맵에서 여러 알고리즘 성능 비교
- `MapVersion` 변경(맵 편집·재생성) 시 자동으로 기록 초기화
- 컬럼: 알고리즘명 / 계산시간(ms) / 방문 노드 수 / 피크 메모리 / 이동 칸 수

#### AlgorithmBenchmark.cs — 일괄 비교 벤치마크
- 버튼 한 번으로 7종 알고리즘을 동일 맵·동일 시작·목표점에서 일괄 실행
- 방문 수 / 경로 길이 / 시간을 표로 출력, 각 지표의 최우수 항목에 `*` 표시
- `PathFinderAI.RunAllAlgorithms`를 호출하여 결과 수집

## 5. 구현된 알고리즘 상세

| 알고리즘 | 자료구조 | 우선순위 기준 | 최단 경로 보장 | 특징 |
|----------|----------|---------------|:---:|------|
| **A\*** | MinHeap | F = G + H (동률 시 H) | O | 휴리스틱+실제 비용 결합, 가장 효율적 |
| **Dijkstra** | MinHeap | G 비용 | O | 모든 방향 균등 확산, 최단 보장 |
| **BFS** | Queue | (FIFO) | O | 가중치 없는 최단 경로, 물결 확산 |
| **DFS** | Stack | (LIFO) | X | 한 방향 깊이 우선, 최단 미보장 |
| **Greedy Best-First** | MinHeap | H 비용 | X | 목표 방향만 추종, 빠르나 최단 미보장 |
| **D\*** | MinHeap | G 비용(역방향) | O | 목표에서 역방향 Dijkstra로 비용 맵 생성 (Stentz, 1994) |
| **D\* Lite** | List(정렬) | key1/key2 | O | 역방향 증분 A*, rhs/g 이중 비용 (Koenig & Likhachev, 2002) |

### 알고리즘별 구현 특징

- **A\* / Dijkstra**: MinHeap 우선순위 큐 사용, 비용 갱신 시 재삽입 후 꺼낼 때 확정 노드는 건너뛰는 **lazy deletion** 방식
- **A\* 휴리스틱**: 맨해튼 거리(4방향 이동에 적합)
- **D\***: 목표점에서 역방향 Dijkstra로 전체 비용 맵을 만든 뒤, 비용 기울기(gCost 감소 방향)를 따라 경로 추출
- **D\* Lite**: 모든 노드를 무한대로 초기화하고 목표의 rhs를 0으로 설정, key1(=min(g,rhs)+h) / key2(=min(g,rhs)) 이중 키로 정렬하며 overconsistent/underconsistent 처리
- **D\* / D\* Lite 공용**: `ExtractGradientPath`로 비용 기울기 추종, 순환 감지 포함

### 성능 지표 정의

| 지표 | 의미 | 해석 |
|------|------|------|
| 계산 시간(ms) | Stopwatch로 측정한 순수 계산 시간 | 낮을수록 빠름 |
| 방문 노드 수 | 탐색 중 확정한 노드 수 | 낮을수록 탐색 효율적 |
| 최대 메모리(피크) | 열린 목록의 최대 크기 | 낮을수록 메모리 효율적 |
| 경로 길이(이동 칸) | 최종 경로의 이동 단계 수 | 낮을수록 경로 품질 높음 |

## 6. 상호작용 시나리오 (사용 흐름)

1. **맵 준비** — 마우스 드래그로 벽을 직접 그리거나, 랜덤 맵 생성 버튼으로 자동 생성. 시작·목표점을 원하는 위치로 이동.
2. **알고리즘 선택** — 드롭다운 또는 버튼으로 7종 중 하나 선택.
3. **탐색 실행** — Move 버튼 → 탐색 과정 애니메이션(방문 노드 확산)이 재생된 후 경로가 파란색으로 강조되고, Player가 경로를 따라 이동.
4. **성능 확인** — 통계 패널에서 방금 실행한 탐색의 지표 확인. 여러 알고리즘을 바꿔 실행하면 대시보드에 누적 비교.
5. **일괄 비교** — 벤치마크 버튼으로 7종을 한 번에 실행하여 표로 최우수 항목 확인.
6. **동적 재탐색 실험** — Player 이동 중 새 벽을 그리면 즉시 경로가 재계산되어 우회.

## 7. 기술적 특징 및 설계 결정

- **머티리얼 인스턴스 방지**: `MaterialPropertyBlock`으로 셀 색상을 설정하여 수백 개 셀에서도 머티리얼 인스턴스 폭증 없이 채색
- **렌더 파이프라인 무관 동작**: `_BaseColor`(URP)와 `_Color`(Built-in) 양쪽에 색상 설정
- **코루틴 트리 통합**: 탐색 애니메이션을 `StartCoroutine`으로 분리하지 않고 직접 `yield`하여, Stop 시 상위 코루틴 하나만 멈춰도 애니메이션·이동이 함께 중단되도록 설계
- **우선순위 큐 최적화**: 매 반복 `List.Sort`(O(n log n)) 대신 MinHeap(O(log n))으로 성능 개선
- **이벤트 기반 디커플링**: `OnSearchCompleted` 이벤트로 대시보드와 탐색 코어를 느슨하게 결합
- **변경 감지 플래그**: `WallsDirty`·`MapVersion`으로 폴링 기반 변경 감지, 재계산·초기화 트리거

## 8. 코딩 컨벤션 (프로젝트 규칙)

- 모든 스크립트는 `Assets/Scripts/` 하위에 용도별 폴더로 분류
- 함수명 PascalCase, 변수명 camelCase, 인스펙터 노출은 `[SerializeField]`
- 모든 변수·함수 선언부 위에 한국어 주석, 스크립트 최상단에 `//TODO:` 로 사용처·기능 명시
- 디자인 패턴 적용 시 TODO 아래 명시
- 예외 처리는 `Debug.Log` / `Debug.LogWarning` / `Debug.LogError` 사용
- 미사용 변수·함수는 삭제하지 않고 `//FIXME:` 주석으로 표기
